Interning at Farsounder

Farsounder is a company that makes sonars that generate a 3-D map of the ocean floor in front of the ship they're installed on. I interned there for around a year, and worked in both C++ and Python. Although I learned C++ in school, I had never written anything in Python before.

I learned how to use Python via my first project at the internship. I wrote various scripts that created a hash-based tag support system that functioned as a lookup tool for the large collection of internal data playbacks. Later in the internship, I expanded on my Python knowledge to work on a research project regarding the feasibility of using the forward-looking sonar for depth approximation. This involved creating visualization tools with tkinter and matplotlib to graph the various depths the forward looking sonar recorded, and then, using tools like numpy, match them against a straight-down only depth finder for accuracy. The main hurdle with this project was correlating the ship's path (which provided the straight down measurement) with the array of data taken from in front of the ship.

I also improved my C++ skills there, it being my first experience with C++ out of a classroom. I completed two separate projects with C++ during my internship. First I implemented improvements to the C++ map based overlay. This involved redesigning the look of how the system displayed the ships recent path. It was my first exercise in digging into a deep codebase to find exactly where the logic I needed to change lived. Once I tracked that down, changing how the line was drawn was pretty easy. With my newly learned skill of "code spelunking" I moved on to the next portion. Using some trigonometry, I created a feature that tracked where on the map the mouse cursor was, and then calculated the distance the cursor was from the bow of the ship, and at what angle it was.

For my final project of the internship, I implemented a custom shortcut key feature. This project used the RegisterHotKey function in C++ to allow the user to define their own shortcuts in an .ini file, and then used ZeroMQ as a message queue to relay the captured hot keys to their corresponding functions in the program (such as zoom, or change modes or themes).

Work at Computer Associates

Computer Associates makes ERP software for millwork companies, lumber yards and door shops. While there I developed in a language call PxPlus. Its a language influenced by Business Basic and boasts the benefits of being easy to use to create GUI screens, its own built in database and it works cross platform. Its also a language I had never heard of when I started my new job!

I started working, right out of college, in the customer support department, where I was responsible for taking customers error reports, often working directly with the users, in order to track down the origins, and then fix bugs in the software. Sometimes the fixes required altering the client's code base, other times it required reconstructing corrupt data. This was a great way to learn both how the industry worked, and learn how to use a language I'd never encountered.

After about a year, I was promoted to become a consultant for the Millwork team. In this role I was responsible for actually developing new software features for clients. I mainly worked directly with clients to customize the software package to better fit their daily needs. The biggest project I worked on, however, lived on the web.

One of its the other nice, newer features, PxPlus allowed for a developer to easily convert an existing code base to be used on the web. One of our clients wanted to create a web store for their business, and I was assigned to help out on the execution. This involved creating a few new displays to interact with the client's database, allowing for their customers to make purchases directly, as well as designing the CSS to fit what the client wanted the site to look like.

This is my rendition of a Fibonacci Clock. Its a cool way to tell time, based on the Fibonacci sequence. small squares are worth 1 each. Then, the values of the squares increase based on size (2,3,5). To tell time you add up the squares. If a square is blue, it counts towards the hours. If it is green, it counts towards minutes. A red square counts for both hours and minutes. To figure out the time, sum the hours and minutes. Minutes need to be multiplied by 5 after they're summed (so it is only accurate within 5 minutes).
